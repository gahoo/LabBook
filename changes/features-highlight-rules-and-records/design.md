# 违规查询与惩罚联动功能设计方案 (Design Specification)

## 1. 预约码呈现设计 (Booking Code Display)
**设计思路：** 不新增独立的表格列，以此保持表格整体视觉的紧凑性。
**呈现位置：** 在上方“违规记录”列表中，直接嵌入到【关联单据】或核心信息区域（例如：仪器名称下方）。
**视觉样式：** 
- 采用低侵入性的排版：`text-xs font-mono text-neutral-400`。
- 格式示例：`预约码: A9B8C7F0`

## 2. 触发规则的呈现维度 (Representation of Triggered Rules)
**核心思想（重中之重）：** 
**绝对保持上方违规记录表格的极简。不需要为每条违规记录添加任何冗余的标签、微标或其他常驻的静态标识。**
- **如何体现用户触发了规则？**：我们将这种表现权完全下放给底部的【活跃限制与惩罚】表格。底部表格**本身**就是用户“已触发规则”的直观仪表盘。管理员只需要看向底部的列表，哪些规则高亮，就能清楚地知道查询结果对应的用户身上已经挂上了哪些惩罚规则。

## 3. 点击溯源互动体验 (Click-to-Highlight Interaction)
**设计思路：** 让底部的规则列表成为强力的“指令来源”，通过互动来建立上下两张表之间的关联。
**具体交互流程：**
1. **指令发起（点击下方的惩罚规则）**：
   下方的【活跃限制与惩罚】表格行变为可交互项（具有 `cursor-pointer` 和轻量的 `hover:bg-neutral-50` 效果）。管理员只需**点击某一具体的规则行**。
2. **聚光灯响应（高亮上方对应的肇事记录）**：
   系统根据这则惩罚背后的 `contributing_ids`，在上方【违规记录】表格中产生即时变化：
   - **肇事记录**：高亮显示（例如加入特定的背景色块、提亮的侧边框等），清晰表明“正是这几条记录导致了本条规则的触发”。
   - **无关记录**：不属于该规则构成的其他违规记录会迅速被按暗，降低透明度至 40%（`opacity-40 grayscale-[10%]`），从视觉上被推向背景。
3. **取消聚焦**：再次点击底部同个规则行，或者点击空白处，上方表格瞬间重置回最初始干爽、完整的常规浏览模式。

## 涉及修改文件
- `src/pages/ViolationQuery.tsx`: 违规与申诉查询的主页面。
- `server.ts`: 更新获取查询相关接口数据的逻辑。
